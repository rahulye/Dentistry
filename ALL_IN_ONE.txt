GITHUB COMMANDS

feat → New feature 				(ex: feat: add calculator display screen)

fix → Bug fix 					(ex: fix: correct division by zero error)

docs → Documentation changes only 		(ex: docs: update README with usage instructions)

style → Code style/formatting, no code logic change 						(ex: style: format CSS file)

refactor → Code change that improves structure but doesn’t add features or fix bugs   (ex: refactor: simplify calculation function)

test → Adding or updating tests 					(ex: test: add unit test for addition)

chore → Maintenance tasks like updating dependencies, configs, or CI/CD    (ex: chore: update npm packages and deployments)



feat → 		New feature
fix → 		Bug fix
docs → 		Documentation changes
style → 	Formatting or UI changes (no logic change)
refactor → 	Code improvement without new features
test → 		Adding or updating tests
chore → 	Maintenance tasks (configs, dependencies)

-------------------------------------------------------------------- STATUS CODE ---------------------------------------------------------------------------

HTTP STATUS CODES

2xx → SUCCESS
200 OK              → Request success
201 Created         → Data created (POST)
204 No Content      → Success, no response body (DELETE)



4xx → CLIENT ERROR
400 Bad Request     → Invalid input     / missing fields
401 Unauthorized    → Not logged in     / token missing or expired
403 Forbidden       → Logged in but no permission
404 Not Found       → Resource not found
409 Conflict        → Duplicate data     (email already exists)

5xx → SERVER ERROR
500 Internal Server Error   → Something broke on server
